Tutorial scikit-learn¶

scikit-learn uma biblioteca python de cdigo aberto utilizada para criao de modelos de machine learning. Atualmente uma das principais bibliotecas utilizadas na cincia de dados, isso ocorre porque o scikit tem uma grande variedade de algoritmos para diversos tipos de modelo e tem uma quantidade de exemplos e documentos para esses modelos.

Introduo para machine learning - O machine learning um subcampo de inteligcia artificial que tem como principal abordagem a de aprender atravs da experincia para encontrar padres em conjunto de dados. Um exemplo do machine learning no nosso dia a dia o algoritmo de sugestes de filmes do netflix, o indentificador de spam no e-mail.

Existem trs principais categorias de machine learning: supervisionado, no-supervisionado e por reforo.

supervisionado - O supervisionamento feito quando a partir de um conjunto de dados rotulados previamente definido, deseja-se encontrar uma funo que seja capaz de predizer rtulos desconhecidos.

no-supervisionado - Nesse tipo de aprendizagem o conjunto de dados utilizado no possui nenhum tipo de rtulo. O objetivo desse tipo de aprendizagem descobrir similaridades entre os objetos analisados a fim de detectar similaridades e anomalias.

Reforo - Nesse tipo de aprendizagem a mquina tenta aprender qual a melhor ao a ser tomada a partir das circunstncias em que essa ao ser executada

Conceitos importantes¶

overfitting - Ocorre quando o modelo est bem ajustado aos dados de treinamento, mas no generaliza to bem.

underfitting - Ocorre quando o modelo muito simples para estrutura de dados subjacentes, ou seja o modelo no generalizava tanto os dados de treinamento como os de teste

Divisão do Dataset¶

Diviso do Dataframe - Sempre que se construir um modelo importante que se separe o DataFrame em pelo menos duas partes treino e teste, para que se garanta que o modelo no ficou viciado nos dados aps o seu treinamento .Normalmente se separa na proporo de 70% pra treino e 30% pra teste, mas isso fica a escolha da pessoa.

No Scikit h um mtodo que separa o dataset de maneira mais simples, mas tambm possvel de ser feito com pandas

Pré-processamento¶

Sklearn.preprocessing - O pr-processamento nada mais do que o processo de preparao, organizao e estruturao dos dados.

Variveis numricas - Nos dados numricos so utilizadas tcnicas para reescalar os dados, pois no dataset utilizado pode ter diversos valores em escalas diferentes e essas diferenas podem prejudicar na qualidade do treino do modelo.

standardScaler - reescala as features removendo a mdia (mdia = 0) e tornando o desvio padrao igual a 1. Desta forma, as features so paradronizadas, tornando-as mais facis manejveis para nossos modelos.

from sklearn.preprocessing import StandardScaler

scaler=StandardScaler()

train_s=scaler.fit_transform(x_train)

test_s=scaler.transform(x_test)

MinMaxScaler - Reescala cada coluna de forma individual em um determinado intervalo que normalmente entre 0 e 1 mas se tiver dados com valores negativos pode variar entre -1 e 1. Colocar os dados na mesma escala faz com que os modelos trabalhem de forma mais eficiente.

from sklearn.preprocessing import MinMaxScaler

scaler=MinMaxScaler(feature_range=(0, 1))

train_s=scaler.fit_transform(x_train)

test_s=scaler.fit_transform(x_test)

RobustScaler - Diferente do MinMax e do Standard o Robust atua com a combinao dos quartis e isso nos garante um bom controle dos outliers. No entanto, ele no impede que tenha outliers mas reduz o seu impacto.

from sklearn.preprocessing import RobustScaler

scaler=RobustScaler()

train_s=scaler.fit_transform(x_train)

test_s=scaler.fit_transform(x_test)

Variveis Categricas - Nos dados categricos se utiliza mtodos de pr-processamentos para transform-los em dados numricos pois existe diversos algoritmos que no conseguem trabalhar com dados categricos. Em problemas de classificao muito comum encontrar na varivel independente dados categricos, um exemplo disso em fazer classificador de cores que na varivel independente tem como cores vermelho, azul e verde e utiliza-se mtodos de pr-processamentos para transforma-los em nmericos, retornando-os como [0,1,2].

Para tratar de variveis categricas usada a combinao dos mtodos LabelEnconder e OneotEnconding. O Label ir transformar os nomes em nmeros e depois utiliza-se o OneHotEncoder para transformar esses nmeros em novas colunas do dataset.

aplicando o LabelEncoder e OneotEnconding:from sklearn.preprocessing import LabelEncoder,OnehotEncoder

label=LabelEncoder()

ohe=OneHotEncoder(categorical_features = [0])

x_l=label.fit_transform(x)

Modelos¶

Regresso - tem como objetivo prever o valor numrico de algo especfico como o valor da ao de alguma empresa. Para os modelos de regresso irei mostrar a regresso linear que o modelo mais simples. Existem diversos modelos da regresso linear que so bem mais complexos como a floresta aleatria.

Regresso linear - A equao da regresso linear estima uma condicional de varivel y, com os dados de uma varivel x, essa equao indica os pontos fortes dessa relao e quantifica este relacionamento.

Aplicando a regresso: from sklearn.linear_model import LinearRegression

lr.fit(x_train,y_train)

lr.coef_ - para observar os coeficientes

lr.intercept_ - ponto de intercepo no eixo y

lr.predict(x_test)- observar o que foi previsto pelo modelo

Classificao - Tem como objetivo prever uma classe, dentro das possibilidades limitadas existentes. Nesse tutorial iremos mostrar a regresso logstica que um dos modelos mais simples para problemas de classificao.

Regresso Logstica - A regresso logstica estima a probabilidade a ocrrncia de uma classe a partir de observaes dos dados de x. Normalmente esse modelo usado para problemas de duas classes mas tambm pode ser usados para problemas de multiclasses.

Aplicando a regresso: from sklearn.linear_model import LogisticRegression

lg.fit(x_train,y_train)

validação cruzada¶

A validao cruzada k-fold consiste em dividir o dados de treino em k partes. Ultiliza-se as k-1 partes para ajustar-se ao modelo usado e a parte restante para a validao. Esse processo repetido k vezes e, em seguida, os resultados so combinados obtendo a mdias dos erros obtidos. Esse mtodo utilizado para garantir que no esteja ocorrendo sobreposio no modelo e para medir a capacidade de predio do modelo.

aplicando a validao cruzada: from sklearn.model_selection import cross_val_score

cv=cross_val_score(modelo,x_train,y_train,cv=(quantidade de k),scoring='mtrica utilizada')

cv.mean()- ir mostrar a pontuao mdia

accuracy_score - proximidade entre o valor previsto e o valor real.
